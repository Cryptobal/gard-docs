# Architecture Decision Records (ADR)

Esta carpeta contiene los **Architecture Decision Records** (ADR) del proyecto GARD DOCS.

## ¿Qué es un ADR?

Un ADR es un documento que captura una decisión arquitectónica importante junto con su contexto y consecuencias.

## Formato Recomendado

Cada ADR debe seguir esta estructura:

```markdown
# ADR-XXX: Título de la Decisión

**Estado:** [Propuesta | Aceptada | Rechazada | Obsoleta]  
**Fecha:** YYYY-MM-DD  
**Autores:** Nombre(s)

## Contexto

¿Cuál es el problema o situación que requiere una decisión?

## Decisión

¿Qué decisión se tomó?

## Alternativas Consideradas

1. **Opción 1:** Descripción
   - Pros: ...
   - Contras: ...

2. **Opción 2:** Descripción
   - Pros: ...
   - Contras: ...

## Consecuencias

¿Qué implicaciones tiene esta decisión?

- ✅ Positivas:
- ⚠️ Negativas:
- 🔄 Trade-offs:

## Referencias

- Links a documentación relevante
- Discusiones relacionadas
```
